# 安全指南

## 认证安全

### JWT 令牌
- 生产环境必须设置强随机 `JWT_SECRET_KEY`
- 建议设置合理的过期时间（默认 1 小时）
- 令牌使用 HMAC-SHA256 签名

### 密码存储
- 使用 PBKDF2-SHA256 + 随机 Salt，迭代 260000 次
- 每次哈希使用不同 Salt，防止彩虹表攻击

### API 密钥
- API 密钥存储时仅保存哈希值，明文密钥只在创建时显示一次
- 建议定期轮换 API 密钥

---

## 访问控制

### 限流防护
- 用户级别：每 60 秒最多 10 次操作（可配置）
- IP 级别：每 60 秒最多 100 次请求
- 防暴力破解：连续失败 5 次后锁定 15 分钟

### 黑名单
- 管理员可将恶意用户加入黑名单
- 黑名单用户无法提交报告

---

## 数据加密

### 传输安全
- 与 Telegram 的所有通信通过 HTTPS 进行
- Webhook 模式下建议使用 HTTPS

### 存储加密
- 敏感数据可使用 `EncryptionService` 加密后存储
- 生产环境必须设置 `ENCRYPTION_KEY` 环境变量
- 使用 AES-256-GCM（需安装 cryptography 包）

---

## 审计日志

系统记录所有关键操作，包括：
- 用户操作（提交报告、查询等）
- 管理员操作（审核、封禁等）
- 认证事件（登录成功/失败）
- 系统事件（启动、错误等）

---

## 最佳实践

1. **定期备份数据库**（启用自动备份）
2. **监控异常登录**（查看审计日志）
3. **及时更新依赖**（防止已知漏洞）
4. **最小权限原则**（限制数据库用户权限）
5. **使用环境变量**（不要在代码中硬编码密钥）
